# Smart Policing Demo Video Script
## Opening And Module Introductions

> 说明：以下内容为 7 分钟左右演示视频的讲解稿，包含 1 段开场总介绍和 5 个模块介绍。正文按配音口径撰写，可根据你已录好的画面节奏做少量删改。

### 1. Opening

各位领导、各位评委，大家好。  
本项目围绕市局“1393”指标体系中“高效能场景”这一目标进行建设，立足基层、面向实战，核心出发点就是把民警从重复、繁琐、低效率的人工筛查工作中解放出来，让机器先跑、数据先筛、系统先判，民警重点做复核和处置。  

过去，在海量图片、现场视频、重点人员核查、后续任务流转等环节，往往存在人工翻看工作量大、线索发现不及时、身份核验链条长、下发处置依赖手工整理等问题。针对这些痛点，我们尝试把目标检测、人脸识别、自动标注、模型训练和任务派发等新技术能力，真正落到基层警务的一个工作台上，形成实战可用、流程闭环、持续迭代的智能研判平台。  

目前，系统共构建了 5 个模块，也就是 5 个工作标签页，分别是：数据检测与研判、现场素材研判、人脸识别与人员核验、模型自训练、任务下发。  

整个项目的价值，不是单点做“识别”，而是把“从海量图像中发现违法行为线索，到进一步核验人员身份，再到自动生成任务、短信提醒和后续流转”串成一条完整链路，形成从发现、研判、核验到处置提醒的全流程闭环。可以说，这套系统既提升了线索发现效率，也提升了基层警务工作的标准化、连续性和可复制性。

---

### 2. Module One: Data Detection And Analysis

#### Technical Summary

第一个模块是“数据检测与研判”。  
这个模块主要面向历史图片数据，技术上采用的是“Oracle 数据库取图 + 多线程下载 + YOLO 目标检测模型批量推理”的方式，对指定时间段内的图片进行自动筛查。系统既支持通用开放词表模型，也支持围绕专项场景训练的专用模型，可以根据不同业务需求切换模型和参数，并将命中结果自动归集、打包输出。

#### Practical Value

它解决的实际问题，是过去基层民警面对历史数据时“图片量大、翻看慢、容易漏”的问题。  
通过这个模块，原来需要人工逐张翻看的工作，现在可以先由系统完成第一轮批量筛查，把疑似违法行为图片快速提取出来，把“海量数据”压缩成“少量重点线索”。这样既节省大量时间，也让后续研判更有针对性，为专项整治和日常巡查提供了稳定、高效的技术支撑。

#### Narration

现在看到的是系统的第一个模块，数据检测与研判。这个模块主要针对数据库中的历史图片数据开展批量筛查。系统可以按照时间范围自动调取图片，再通过目标检测模型进行统一识别，把疑似违法行为线索快速筛出来。过去这类工作主要依赖人工逐张查看，不仅耗时耗力，而且容易遗漏。现在通过机器先筛查、人工再复核的方式，可以显著提升线索发现效率，把民警从大批量重复劳动中解放出来。

---

### 3. Module Two: On-site Material Analysis

#### Technical Summary

第二个模块是“现场素材研判”。  
这个模块主要处理基层上传的图片压缩包和现场视频素材。技术上支持 ZIP 图片包解析、视频抽帧、统一模型推理和结果回传，能够把不同来源、不同格式的现场素材统一纳入同一套识别流程中。对于视频，系统会按照设定间隔自动抽帧检测，避免人工拖动进度条逐段查找。

#### Practical Value

它解决的是一线实战中“素材在手但研判慢”的问题。  
无论是执法记录、现场取证，还是群众举报、临时汇集的视频和图片，只要上传到系统，就可以迅速进入自动分析流程。这样做的意义在于，让基层民警在面对突发线索和现场素材时，能够更快得到研判结果，更快锁定重点画面和关键对象，提升现场处置和后续研判效率。

#### Narration

第二个模块是现场素材研判。针对基层经常遇到的图片包、视频片段等非结构化素材，系统支持直接上传，并自动完成图片解析、视频抽帧和批量识别。以往这类工作往往依赖人工反复播放、暂停、截图，不仅效率低，而且容易错过关键画面。现在系统可以先把素材中的疑点自动提取出来，帮助民警更快从现场资料中找到有效线索，实现从“看素材”向“用系统筛素材”转变。

---

### 4. Module Three: Face Recognition And Identity Verification

#### Technical Summary

第三个模块是“人脸识别与人员核验”。  
这个模块建立在前端识别结果之上，进一步叠加了人脸检测、特征提取和人脸库比对能力。系统可以对命中的结果图片进行人脸识别，从中提取有效人脸，再与人脸库进行相似度检索，输出候选身份信息。同时，系统支持人脸库同步、重建和结果回流，为后续任务下发提供结构化身份基础。

#### Practical Value

它解决的是“发现了线索，但还要继续查人”的问题。  
过去从发现可疑行为到核验人员身份，中间通常需要多系统切换、多轮人工比对。现在系统将行为识别结果和人员身份核验衔接起来，在保留人工审核空间的前提下，尽可能缩短从“发现问题”到“锁定对象”的时间。这样不仅提升了处置效率，也为后续派发、提醒、落地核查打通了关键一环。

#### Narration

第三个模块是人脸识别与人员核验。前两个模块解决的是“发现线索”的问题，而这个模块解决的是“找到人”的问题。系统会对命中结果中的人脸进行自动提取和比对，给出候选身份信息，为民警进一步核验提供依据。过去发现线索以后，往往还需要再人工切换到其他系统继续查找对象。现在通过把行为识别和身份核验衔接起来，可以明显缩短从发现线索到锁定重点人员的时间，提高研判效率和工作连续性。

---

### 5. Module Four: Model Self-training

#### Technical Summary

第四个模块是“模型自训练”。  
这个模块围绕“数据集管理、自动标注、训练任务、模型注册、模型切换”展开。系统支持把前面识别得到的高价值结果图片回流成数据集，支持人工修订标注，也支持利用现有模型进行自动标注，在此基础上继续训练专项模型。训练完成后，模型可以注册入库，并按槽位切换到对应业务场景中，实现模型能力的持续迭代。

#### Practical Value

它解决的是“模型一旦部署就很难再成长”的问题。  
基层警务场景变化快，如果只依赖一次性训练好的模型，后续很容易出现适配度下降、识别效果不够贴近实战的问题。通过这个模块，系统可以把实战中不断积累的新样本持续沉淀下来，逐步形成“实战发现数据、数据反哺模型、模型再服务实战”的正向循环，让系统不是一次性工具，而是可以不断成长的能力平台。

#### Narration

第四个模块是模型自训练。这个模块是整个平台持续成长的关键。系统不仅能做识别，还能把识别过程中沉淀下来的高价值样本重新整理成数据集，结合人工修订和自动标注，继续训练更贴近本地实战的新模型。训练完成后，还可以直接纳入系统使用。这样一来，平台就形成了“越用越懂业务、越用越贴近实战”的能力闭环，不断提升专项识别的准确性和针对性。

---

### 6. Module Five: Task Dispatch

#### Technical Summary

第五个模块是“任务下发”。  
这个模块的核心能力，是把前面识别和核验得到的结构化结果，自动整理成可下发、可提醒、可追踪的任务数据。系统支持待派发队列管理、派发内容预览、任务推送和短信提醒等功能，把原本分散在多个步骤中的人工整理、复制粘贴和重复录入工作，压缩成一套标准化流程。

#### Practical Value

它解决的是“线索找到了，但流转慢、落地慢”的问题。  
很多时候，真正影响效率的不是发现线索本身，而是后续如何转成任务、怎么提醒到人、怎么形成闭环。这个模块的价值就在于，能够把识别结果和后续动作连接起来，让系统不仅会“看”，还会“推”和“转”，推动线索更快进入处置环节，减少人工整理过程中的时间损耗和信息遗漏。

#### Narration

第五个模块是任务下发。当前面已经完成行为识别和身份核验之后，系统会进一步把相关结果自动整理成待下发任务，并支持短信提醒等方式，推动后续核查处置。过去从发现问题到形成任务，中间往往还有大量手工整理和重复录入工作。现在系统将识别、核验、派发串联起来，把“发现线索”进一步转化为“推动处置”，真正实现从前端感知到后端流转的闭环。

---

### 7. Closing Suggestion

整个项目的核心，不是简单增加了几个功能模块，而是围绕基层警务实战，把“数据筛查、素材研判、身份核验、模型成长、任务流转”连接成了一套完整工作链条。  
它既体现了新技术在公安业务场景中的实际应用，也体现了系统建设从“能演示”向“能落地、能使用、能持续优化”的转变。下一步，我们也希望继续结合基层反馈，不断优化模型效果、丰富业务规则、完善闭环流程，真正让技术更懂业务，让系统更贴近实战。
