# Smart Policing Demo Video Script
## Opening And Module Introductions

> 说明：以下内容为 7 分钟左右演示视频的讲解稿，包含 1 段开场总介绍和 5 个模块介绍。正文按配音口径撰写，可根据已录好的画面节奏做少量删改。

### 1. Opening

各位领导、各位评委，大家好。

本项目依托市局"1393"指标体系，围绕"高效能场景"这一核心目标进行建设。项目的出发点很明确：基层民警长期面临海量图片逐张翻看、现场素材人工研判、人员身份多系统核查、任务流转手工整理等繁重工作，效率低、容易漏、难以形成系统性闭环。对此，我们尝试将目标检测、人脸识别、自动标注、模型自训练、任务自动派发等新技术能力，整合到一个面向基层实战的统一工作台上，让机器替代人工完成高重复性环节，让民警把精力集中在复核与处置上。

目前系统共建设了 5 个功能模块，分别是：数据检测与研判、现场素材研判、人脸识别与人员核验、模型自训练、任务下发。

整个项目的核心价值，不在于单点的"识别"能力，而在于完整的链路闭环：系统能够从海量图像中精准发现违法行为线索，进一步对涉案人员进行身份核验，再自动生成任务、触发短信提醒、推动后续处置流转真正实现从发现、研判、核验到处置的全流程闭环。这套系统既大幅提升了线索发现效率，也推动了基层警务工作的标准化与可持续运转。

---

### 2. Module One: Data Detection And Analysis

#### Technical Summary

第一个模块是"数据检测与研判"。

技术上，本模块采用"Oracle 数据库按时间范围取图  多线程并发下载  YOLO 目标检测模型批量推理"的方式，对历史图片数据进行自动筛查。系统同时支持通用开放词表模型（YOLO-World）和针对专项场景自训练的专用模型，可按业务需求灵活切换模型与参数，命中结果自动归集并打包输出。

#### Practical Value

它解决的核心问题，是基层民警面对海量历史图片时"图片量大、翻看慢、容易漏"的痛点。

通过本模块，原本需要人工逐张翻看的工作，由系统完成第一轮批量筛查，将海量数据快速压缩成少量重点线索，让后续研判更有针对性。无论是专项整治还是日常巡查，都能在极短时间内完成大范围图片的初步过滤，显著降低人工工作量。

---

### 3. Module Two: On-site Material Analysis

#### Technical Summary

第二个模块是"现场素材研判"。

技术上，本模块支持 ZIP 图片包解压解析、MP4/AVI/MOV 视频自动抽帧（按设定间隔逐帧提取）、统一模型批量推理与结果回传，能够将不同来源、不同格式的现场素材纳入同一套识别流程进行统一处理。

#### Practical Value

它解决的是一线实战中"素材在手但研判慢"的问题。

无论是执法记录仪视频、现场取证图片，还是群众举报的视频片段，上传后即可进入自动分析流程。系统自动代替人工完成反复播放、暂停、截图、逐帧查找的繁琐过程，帮助民警更快从现场素材中锁定关键画面与重点对象，实现从"人工看素材"向"系统筛素材"的转变。

---

### 4. Module Three: Face Recognition And Identity Verification

#### Technical Summary

第三个模块是"人脸识别与人员核验"。

技术上，本模块在前两个模块的检测结果之上，叠加了人脸检测、深度学习特征提取、人脸库相似度检索等能力。系统对命中结果图片自动提取有效人脸，与已建立的人脸库进行比对，输出候选身份信息。同时支持人脸库的同步、增量更新与重建，为后续任务下发提供结构化的身份数据基础。

#### Practical Value

它解决的是"发现了线索，但还要继续查人"的效率瓶颈。

过去从发现可疑行为到完成人员身份核验，通常需要在多个系统之间来回切换、多轮人工比对，耗时耗力。本模块将行为识别结果与身份核验直接衔接，在保留人工审核空间的前提下，大幅缩短从"发现问题"到"锁定对象"的时间，为后续处置派发打通了关键一环。

---

### 5. Module Four: Model Self-training

#### Technical Summary

第四个模块是"模型自训练"。

技术上，本模块围绕"数据集管理  自动标注  训练任务调度  模型注册  模型槽位切换"构建完整的训练闭环。系统支持将前序模块识别得到的高价值结果图片回流为训练数据集，支持人工修订标注或利用现有模型进行自动预标注，在此基础上继续训练专项模型，训练完成后注册入库并按槽位灵活切换至对应业务场景。

#### Practical Value

它解决的是"模型部署后难以持续成长"的问题。

基层警务场景复杂多变，若长期依赖一次性训练的固定模型，识别效果会逐渐与实战脱节。本模块让系统能够把实战中持续积累的新样本沉淀为训练数据，形成"实战产生数据、数据反哺模型、模型再服务实战"的正向循环，使平台不是一次性工具，而是具备持续学习、不断贴近本地场景的能力平台。

---

### 6. Module Five: Task Dispatch

#### Technical Summary

第五个模块是"任务下发"。

技术上，本模块将前序识别与核验产生的结构化结果，通过待派发队列管理、派发内容预览、任务自动推送与短信提醒等功能，转化为可下发、可追踪的标准化任务数据，取代了原本依赖人工整理、复制粘贴和重复录入的分散性操作。

#### Practical Value

它解决的是"线索找到了，但流转慢、落地慢"的最后一公里问题。

真正制约处置效率的，往往不是发现线索本身，而是后续如何将线索转化为任务、如何快速提醒到人、如何推动处置形成闭环。本模块将识别、核验、派发贯通为一条流水线，让系统不仅会"发现"，还会"推动"，减少人工整理过程中的时间损耗与信息遗漏，真正实现从前端感知到后端流转的闭环贯通。
