## 一、活动背景与目的

### （一）背景

近期，C市飙车炸街违法行为多发，网络平台（抖音等）出现大量本地相关视频，炫技、结队、改装摩托车出行等内容传播广泛，社会影响恶劣，群众反映强烈。经专项整治，截至目前全市已依法查处涉及飙车炸街的未成年人**20余名**，均依据未成年人相关法规处理，对情节较重者给予行政拘留处罚（系对涉案未成年人可适用的最高行政处罚手段），但飙车炸街活动的未成年人参与问题仍不容忽视。

该类行为具有强示范效应，现实与网络的双重传播使未成年人群体极易跟随效仿，若不能做到早发现、早干预，极易从一般交通违法滑向打架斗殴、聚众滋事等更为严重的违法犯罪。然而，传统执法模式面临以下突出困境：

- **信息量大**：沿线电子警察和视频卡口每日产生数万张抓拍图片；
- **人力不足**：人工逐张筛查不仅效率低，高强度作业还容易造成漏检和误判；
- **链条过长**：从"发现违法图片"到"落地处置"中间环节多，涉及未成年人的线索往往不能得到及时干预。

正是在这一背景下，XX市公安局治安管理支队基础管控中心在参与飙车炸街专项整治工作专班期间，针对以上问题自主搭建了"**猎影哨兵**"智能视觉研判平台，将原来的"人工翻图—人工核身—人工派活"模式，改造为"自动检测—自动比对—自动下发"的一体化工作链路。

### （二）活动目的

借本次警营开放日活动，面向参与的领导、民警和学生，展示数据赋能警务的实战经验，以飙车炸街专项整治中未成年人违法行为的发现与处置工作为切入点，有条件时通过实机演示，使参与人员直观了解：

1. 人工智能技术如何辅助精准发现未成年人违法行为、缩短干预周期；
2. 数据化警务模式在预防未成年人犯罪工作中的实际作用；
3. 基层民警立足实战需求、自主研发应用工具的工作路径与思路。

## 二、未成年人预防犯罪相关模型展示

### （一）飙车炸街与未成年人违法的现实情况

飙车炸街活动的参与者中，未成年人占有相当比例。根据现阶段执法实践，涉案未成年人呈现以下共同特点：

- **年龄段集中**：多为初中至高中阶段在校或辍学学生；
- **违法行为轻重参杂**：既有跟风骑行、凑热闹等轻微情形，也有翘车头、逆行、聚集滋事等安全隐患突出的行为；
- **网络引流明显**：抖音等平台上的相关视频起到了明显的示范和聚集效果，部分未成年人因刷到"本地同龄人"视频而主动加入。

对上述人员若不能做到**早发现、快处置**，轻则酿成交通事故，重则演变为聚众斗殴、寻衅滋事等刑事案件。早期智能预警是未成年人犯罪预防工作的重要一环。

### （二）"猎影哨兵"平台功能说明

该平台由XX市公安局治安管理支队基础管控中心民警于2026年3月自主研发，以深度学习目标检测与图像分类技术为核心，面向内网部署，无需互联网连接。平台实现如下四个核心功能模块的闭环流转：

#### 1. 智能图片筛查（检测模块）

- **数据接入**：可直连卡口/电子警察后台数据库，按时间范围批量拉取抓拍图片，无需人工导出，也支持本地上传ZIP图片包及MP4/AVI/MOV格式视频文件；
- **违法特征识别**：内置"飙车炸街"专项训练模型，针对以下典型违法场景进行自动识别标注：
  - 摩托车翘车头（前轮腾空）
  - 多辆车结队过卡
  - 骑手未佩戴头盔
  - 摩托车多人搭乘
- **运行效率**：单台普通内网办公电脑（无需独立GPU）可达**每分钟处理约500张图片**的识别效率；
- **结果输出**：检测完成后自动生成分类结果打包文件，民警下班前启动任务，次日上班直接下载使用，**无需在岗盯守**。

#### 2. 人脸识别与身份核验（人脸模块）

- 对检测出的违法图片，自动提取骑手面部，调用人脸识别模型进行批量身份比对；
- 结果结构化输出，可快速锁定涉事人员姓名、年龄、户籍等信息，对**未成年人自动优先标注**，便于分类处置；
- 人脸库数据来源为内部业务数据库同步，全程在内网环境中运行，数据不外传。

#### 3. 任务下发与跟踪（下发模块）

- 识别结果自动流入任务下发队列，通过省厅治综任务平台**一键推送至属地派出所**；
- 同步触发短信提醒，跟进处置情况，显著压缩"发现违法—落地处置"的中间时间差；
- 对涉及未成年人的线索可按优先级单独推送，确保公安机关能够第一时间启动训诫、家访、责令父母管教等干预措施。

#### 4. 样本回流与模型持续优化（训练模块）

- 每轮整治周期的检测结果可回流至训练数据集，经标注、预标注和YOLO模型再训练后，识别准确率持续提升；
- 新模型发布后自动替换上线，形成"**发现—处置—样本积累—模型优化—再发现**"的正向闭环，适应不同时期飙车炸街的新型手法和新兴违法场景。

### （三）平台实战成效

"猎影哨兵"平台自2025年10月在飙车炸街专项整治工作专班正式投入应用以来，已协助处理辖区相关卡口图片逾**百万张**，取得以下可量化成效：

| 指标 | 数据 |
|------|------|
| 全市飙车炸街警情 | 接报 **620起**，同比下降 **11.8%** |
| 专项整治攻坚阶段警情降幅 | 同比下降 **37.4%** |
| 查处违法行为 | **7692起**，同比大幅上升 |
| 查处比值（攻坚期） | 跃升至 **72.3**，较整治前提升约 **13倍** |
| 扣留涉案车辆 | **7051辆** |
| 行政拘留 | **437人** |
| 刑事立案 | **8起** |
| 已处置涉案未成年人 | **20余名** |
| 释放专职筛图警力 | **2名**新警从日均手工翻图工作中解放，重新投入研判核查 |

**对未成年人工作的直接意义**：通过精准、快速的违法线索发现和身份核验，使公安机关可在未成年人违法行为初期即介入干预，通过训诫、责令父母管教等措施，及时阻断其进一步滑向更严重违法犯罪的轨迹。

## 三、场景设置

> **注**：本次活动具体场地尚未确定，以下为预设展示方案；若具备条件可按此落实，若现场条件有限，可灵活调整为展板展示或视频播放形式。

### （一）展示区域设置

建议在警营开放日活动现场设置"**智能警务实战展示区**"，划定独立展示区域，面积约15-20平方米，配置如下：

| 设备/物料 | 数量 | 说明 |
|-----------|------|------|
| 演示笔记本电脑 | 1台 | 已完成平台部署，预载脱敏演示数据，提前测试通过 |
| 外接大屏（投影仪或显示屏） | 1套 | 供多名学员同时观看界面操作 |
| 展板/易拉宝 | 1-2块 | 展示平台功能流程图、核心战果数据及飙车炸街情况说明 |
| 宣传资料单页 | 按参观人数准备 | 平台简介及功能说明，便于学员带走回顾 |

### （二）演示流程

每场次演示时长约**15-20分钟**，采用讲解+实机操作方式，可循环开展，具体流程如下：

**第一步：背景引入（约3分钟）**

介绍C市飙车炸街现状，可现场展示典型网络视频截图，说明传统人工筛查面临的困境，引出"猎影哨兵"平台建设的背景和必要性。

**第二步：实机演示（约10分钟）**

1. 打开平台工作台界面，进行"本地上传检测"演示：上传预先准备的演示图片包，触发自动检测；
2. 实时展示推理过程（标注框、违法类别标签、置信度得分），让学员直观感受识别速度；
3. 演示人脸识别环节：从违法图片中提取面部、与库比对、输出身份核验结果（演示用脱敏样本）；
4. 演示任务下发界面：识别结果流入下发队列、一键推送处置单位的完整操作流程；
5. 简要演示未成年人标注优先处置机制。

**第三步：成效展示与互动（约5分钟）**

展示整治成效数据，开放互动环节：重点对比说明"人工翻图"和"系统自动筛查"在效率、准确性、警力占用上的差异，回应提问；针对不同受众可有所侧重——对领导侧重战果数据和推广价值，对一线民警侧重实操减负效果，对学生侧重技术原理和实战思路。

### （三）注意事项

1. 演示全程使用**脱敏数据**，真实案例图片须做人像模糊处理；未成年人原始图片不得展示；
2. 演示设备仅在内网环境中运行，**活动期间不得接入互联网**；
3. 如当日参观人数较多、场次密集，可事先录制系统操作演示视频作为备用，确保讲解流畅连贯；
4. 讲解人员重点突出实战应用逻辑，避免过度技术化解释；面向领导时重点说明应用成效与推广潜力，面向学生时适当增加系统原理说明，确保不同受众均能理解实战价值。

## 四、相关要求和保障

### （一）技术保障

1. **设备准备**：演示使用的笔记本电脑须提前完成平台部署，确认模型文件（飙车炸街专项模型等）正确加载，本地SQLite数据库连接测试通过；
2. **数据准备**：提前制作脱敏演示图片包，建议覆盖翘车头、结队过卡、未戴头盔、多人搭乘各类场景，数量300-500张为宜；完整流程须于活动前进行至少一次全流程演练；
3. **备用方案**：提前录制完整演示操作视频备用，防止现场出现系统异常导致演示中断；
4. **场地电源**：确认展示区域供电稳定，自备延长线，防止设备因电源问题中断。

### （二）人员保障

| 职责 | 人数 | 具体要求 |
|------|------|----------|
| 主讲/演示民警 | 1名 | 熟悉平台操作和业务背景，能结合飙车炸街实战案例进行讲解 |
| 现场协助 | 1名 | 负责场地协调、参观学员引导、资料发放等现场组织事务 |

### （三）数据安全与保密要求

1. 演示使用的所有图片均须事先完成脱敏处理，不得包含可直接识别的未成年人面部或其他个人信息；
2. 演示设备不存储、不传输涉密信息；活动结束后及时清除演示数据；
3. 对平台中涉及具体案件数据、人脸识别接口及对接外部系统等敏感内容，讲解时应适可而止，不做详细展开；
4. 严格落实"最小化展示"原则，只演示必要功能模块，不涉及后台数据库权限操作，不演示真实人员数据查询。

### （四）审批与汇报要求

1. 本方案须经支队主管领导审批同意后方可实施，相关展示内容（含演示数据、讲解口径）须经确认；
2. 活动结束后一周内，由承办部门形成书面工作小结，记录参观学员人数、现场反馈及改进建议，报支队备案；
3. 如涉及媒体采访、警务号发文、现场录像对外发布等情形，须另行按保密规定和宣传审批程序报批，不得自行发布。
